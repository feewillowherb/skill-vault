# Monospecs 快速参考

**版本**: 1.0  
**更新日期**: 2026-05-14  
**用途**: 常用命令和配置速查表

## 快速开始模板

### 最小化配置模板

```yaml
# monospecs.yaml
version: "1.0"
commit_when_archive: true

repositories:
  - path: "repos/my-app"
    url: "https://github.com/your-org/my-app.git"
    displayName: "我的应用"
    icon: "🚀"
```

### 标准项目结构

```
my-project/
├── .gitignore              # 必须包含 repos/
├── monospecs.yaml          # 核心配置文件
├── AGENTS.md               # AI 指导文档
├── openspec/               # 变更管理目录
└── repos/                  # 子仓库目录（Git 排除）
    ├── frontend/           # 子仓库
    └── backend/            # 子仓库
```

## 常用命令

### 初始化与设置

```bash
# 创建新项目
monospecs init my-project

# 验证配置文件
monospecs validate

# 添加到 HagiCode
# 通过桌面应用界面操作
```

### 仓库操作

```bash
# 克隆所有仓库
monospecs clone

# 更新所有仓库
monospecs update

# 检查仓库状态
monospecs status

# 清理未配置的仓库
monospecs clean
```

### 仓库管理

```bash
# 添加新仓库
monospecs add https://github.com/org/new-repo.git

# 移除仓库
monospecs remove repos/new-repo

# 同步配置与实际状态
monospecs sync
```

## 配置文件速查

### 配置项说明

| 配置项 | 类型 | 必需 | 默认值 | 说明 |
|-------|------|------|--------|------|
| `version` | string | ✅ | - | 配置文件版本，当前为 "1.0" |
| `commit_when_archive` | boolean | ❌ | false | 归档时自动提交 specs |
| `repositories` | array | ✅ | - | 仓库配置列表 |
| `repositories[].path` | string | ✅ | - | 仓库相对路径，必须以 `repos/` 开头 |
| `repositories[].url` | string | ✅ | - | Git 仓库 URL |
| `repositories[].displayName` | string | ✅ | - | 仓库显示名称 |
| `repositories[].icon` | string | ❌ | "📦" | 仓库图标（emoji） |

### 命名规范

```yaml
# ✅ 正确的命名
repositories:
  - path: "repos/frontend-service"     # kebab-case
    displayName: "前端服务"             # 简洁中文
    icon: "🌐"                          # 相关 emoji
    
  - path: "repos/user-auth-module"     # kebab-case
    displayName: "用户认证模块"          # 功能描述
    icon: "🔐"                          # 相关 emoji

# ❌ 错误的命名
repositories:
  - path: "repos/FrontEndService"      # 不应使用 PascalCase
    displayName: "前端服务              # 缺少结束引号
    icon: "invalid"                     # 无效 emoji
```

## 故障排除速查

### 常见错误及解决方案

| 错误症状 | 可能原因 | 解决方案 |
|---------|---------|---------|
| 子仓库未显示 | 配置文件语法错误 | 检查 YAML 缩进和语法 |
| Git 操作失败 | URL 错误或无权限 | 验证 Git URL 和访问权限 |
| 配置无效 | 缺少必需字段 | 检查所有必需字段是否存在 |
| 路径错误 | 路径格式不正确 | 确保使用 `repos/` 前缀 |
| 归档失败 | OpenSpec 配置问题 | 检查 `commit_when_archive` 设置 |

### 快速诊断命令

```bash
# 验证配置文件语法
yq eval monospecs.yaml

# 检查 Git 连接
git ls-remote https://github.com/org/repo.git

# 验证目录结构
ls -la repos/

# 检查 Git 状态
cd repos/your-repo && git status
```

## OpenSpec 集成要点

### 变更创建流程

```bash
# 1. 创建变更目录
mkdir -p openspec/changes/feature-x

# 2. 创建变更文件
cd openspec/changes/feature-x
echo "# Feature X" > proposal.md
echo "# Design" > design.md
echo "# Tasks" > tasks.md

# 3. 实现代码变更
cd ../../repos/frontend
# 进行代码修改...

# 4. 归档变更（自动提交 specs）
# 通过 HagiCode 界面操作

# 5. 手动提交代码
cd repos/frontend
git add .
git commit -m "Implement feature x"
```

### 跨仓库变更要点

- **变更范围**: 涉及多个子仓库的变更
- **specs 位置**: 始终在主仓库的 `openspec/` 目录
- **代码位置**: 在对应的子仓库中
- **归档行为**: 自动提交 specs，手动提交代码

## 技术支持

### 遇到问题时的检查顺序

1. **配置文件**: 检查 `monospecs.yaml` 语法和结构
2. **目录结构**: 确认 `repos/` 目录和子目录存在
3. **Git 配置**: 验证 Git URL 和访问权限
4. **工具版本**: 确认使用最新版本的 HagiCode 工具
5. **文档参考**: 查阅官方文档获取最新信息

### 调试技巧

```bash
# 详细输出模式
monospecs --verbose clone

# 干运行模式（不实际执行）
monospecs --dry-run update

# 配置文件验证
monospecs validate --detailed

# 仓库状态详细信息
monospecs status --full
```
