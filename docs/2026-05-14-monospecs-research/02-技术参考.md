## 架构设计

### 系统架构

Monospecs 采用**主从架构**设计，通过配置驱动的多仓库管理：

```
┌─────────────────────────────────────────────────────────┐
│                    主仓库 (Main Repo)                    │
│  ┌────────────────────────────────────────────────────┐ │
│  │  monospecs.yaml (核心配置)                         │ │
│  │  openspec/ (变更管理)                              │ │
│  │  scripts/ (自动化脚本)                             │ │
│  │  AGENTS.md (AI 指导)                               │ │
│  └────────────────────────────────────────────────────┘ │
│                          ↓                              │
│  ┌────────────────────────────────────────────────────┐ │
│  │              repos/ (Git 排除目录)                 │ │
│  │  ┌──────────┬──────────┬──────────┬──────────┐    │ │
│  │  │frontend  │backend   │service-a │service-b │    │ │
│  │  │(独立仓库)│(独立仓库)│(独立仓库)│(独立仓库)│    │ │
│  │  └──────────┴──────────┴──────────┴──────────┘    │ │
│  └────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

### 核心组件

#### 1. 配置解析器 (Config Parser)

**职责**: 解析 `monospecs.yaml` 配置文件

**技术要点**:
- YAML 格式解析
- 配置验证和错误处理
- 默认值填充和类型转换

```yaml
# 配置文件结构模型
MonospecsConfig:
  version: string (必需)
  commit_when_archive: boolean (可选，默认 false)
  repositories: array<Repository> (必需)
  
Repository:
  path: string (必需，相对路径)
  url: string (必需，Git URL)
  displayName: string (必需，显示名称)
  icon: string (可选，emoji 图标)
```

#### 2. 仓库管理器 (Repository Manager)

**职责**: 管理子仓库的生命周期

**核心功能**:
- 仓库发现和识别
- 批量操作支持
- 状态监控和同步

**操作类型**:
```bash
# 基础操作
clone    # 克隆配置的仓库
update    # 更新仓库到最新版本
status    # 检查仓库状态
clean     # 清理未配置的仓库

# 批量操作
batch-clone    # 批量克隆
batch-update   # 批量更新
batch-status   # 批量状态检查
```

#### 3. OpenSpec 集成器 (OpenSpec Integrator)

**职责**: 与 OpenSpec 系统集成

**集成点**:
- 变更创建时读取仓库配置
- 归档时自动提交 specs
- 跨仓库变更路由

**工作流程**:
```
1. 用户创建变更 → OpenSpec 读取 monospecs.yaml
2. 选择目标仓库 → 显示可用子仓库列表
3. 实现代码变更 → 在对应子仓库中修改代码
4. 执行归档操作 → 自动提交 specs 到主仓库
5. 手动提交代码 → 子仓库代码变更独立提交
```

### 关键算法

#### 仓库发现算法

```python
def discover_repositories(config_path: str) -> List[Repository]:
    """
    从配置文件发现仓库
    """
    # 1. 解析配置文件
    config = parse_yaml(config_path)
    
    # 2. 验证配置
    validate_config(config)
    
    # 3. 提取仓库列表
    repositories = []
    for repo_config in config['repositories']:
        repo = Repository(
            path=repo_config['path'],
            url=repo_config['url'],
            display_name=repo_config['displayName'],
            icon=repo_config.get('icon', '📦')
        )
        repositories.append(repo)
    
    return repositories
```

#### 批量操作算法

```python
def batch_operation(repositories: List[Repository], 
                   operation: Callable[[Repository], None]) -> Dict[str, Any]:
    """
    对多个仓库执行批量操作
    """
    results = {
        'success': [],
        'failed': [],
        'summary': {}
    }
    
    for repo in repositories:
        try:
            operation(repo)
            results['success'].append(repo.path)
        except Exception as e:
            results['failed'].append({
                'path': repo.path,
                'error': str(e)
            })
    
    # 生成摘要
    results['summary'] = {
        'total': len(repositories),
        'success_count': len(results['success']),
        'failed_count': len(results['failed'])
    }
    
    return results
```

#### 配置同步检查算法

```python
def check_config_sync(config: MonospecsConfig) -> SyncReport:
    """
    检查配置与实际状态的同步情况
    """
    report = SyncReport()
    
    for repo_config in config.repositories:
        actual_path = repo_config.path
        
        # 检查目录是否存在
        if not os.path.exists(actual_path):
            report.add_missing(repo_config.path)
            continue
            
        # 检查是否为 Git 仓库
        if not is_git_repository(actual_path):
            report.add_invalid(repo_config.path)
            continue
            
        # 检查 URL 匹配
        actual_url = get_git_remote_url(actual_path)
        if actual_url != repo_config.url:
            report.add_mismatch(repo_config.path, 'url', actual_url, repo_config.url)
    
    return report
```

## 性能考虑

### 大规模仓库管理

#### 性能优化策略

1. **并行操作**:
```python
def parallel_clone(repositories: List[Repository]) -> None:
    """
    并行克隆多个仓库以提高性能
    """
    with ThreadPoolExecutor(max_workers=5) as executor:
        futures = [executor.submit(clone_repo, repo) for repo in repositories]
        for future in futures:
            future.result()  # 等待所有任务完成
```

2. **增量更新**:
```python
def smart_update(repositories: List[Repository]) -> None:
    """
    智能更新：仅更新需要更新的仓库
    """
    for repo in repositories:
        if needs_update(repo):
            update_repo(repo)
```

3. **缓存机制**:
```python
class RepositoryCache:
    """
    仓库状态缓存
    """
    def __init__(self):
        self.cache = {}
        self.ttl = 300  # 5分钟缓存
    
    def get_status(self, repo: Repository) -> RepositoryStatus:
        if repo.path in self.cache:
            return self.cache[repo.path]
        
        status = self._fetch_status(repo)
        self.cache[repo.path] = status
        return status
```

### 内存优化

#### 懒加载策略

```python
class LazyRepositoryLoader:
    """
    懒加载仓库配置
    """
    def __init__(self, config_path: str):
        self.config_path = config_path
        self._config = None
    
    @property
    def config(self) -> MonospecsConfig:
        if self._config is None:
            self._config = self._load_config()
        return self._config
    
    def _load_config(self) -> MonospecsConfig:
        # 仅在首次访问时加载配置
        return parse_yaml(self.config_path)
```

#### 插件示例

```typescript
// 仓库验证插件
class RepositoryValidatorPlugin implements MonospecsPlugin {
  name = "repository-validator";
  version = "1.0.0";
  
  validateRepository(repo: Repository): ValidationResult {
    const errors = [];
    
    // 验证 URL 格式
    if (!this.isValidGitUrl(repo.url)) {
      errors.push("Invalid Git URL format");
    }
    
    // 验证路径格式
    if (!repo.path.startsWith("repos/")) {
      errors.push("Path must start with 'repos/'");
    }
    
    return {
      valid: errors.length === 0,
      errors
    };
  }
  
  private isValidGitUrl(url: string): boolean {
    return /^https?:\/\/.+\.git$/.test(url);
  }
}
```

## 技术债务与改进方向

### 已知限制

1. **YAML 解析性能**: 大规模配置文件解析性能有待优化
2. **并行操作控制**: 缺乏细粒度的并发控制
3. **错误恢复**: 操作失败后的恢复机制不够完善
4. **缓存策略**: 缓存一致性需要改进

### 改进建议

1. **性能优化**:
   - 实现增量配置解析
   - 优化大规模仓库操作性能
   - 改进缓存策略

2. **功能增强**:
   - 支持更多版本控制系统
   - 增强错误处理和恢复
   - 改进插件系统

3. **可观测性**:
   - 增强日志和监控
   - 提供性能分析工具
   - 改进调试体验
